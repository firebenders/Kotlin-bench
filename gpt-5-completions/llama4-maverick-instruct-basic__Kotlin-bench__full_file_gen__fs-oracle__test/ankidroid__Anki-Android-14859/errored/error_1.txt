To solve the issue of the display order bug for notes in the Card Browser, we need to make several changes to the code.

### Step 1: Understand the Problem
The problem is that when the user switches to "notes" mode in the Card Browser and changes the display order, it doesn't stick. The order reverts to "by created time" (oldest to newest) after certain actions.

### Step 2: Identify the Relevant Code
The relevant code is in `CardBrowser.kt` and `SortType.kt`. The `CardBrowser` class handles the display and sorting of cards/notes, and `SortType` is an enum that defines the different sorting options.

### Step 3: Analyze the Issue
The issue arises because the sorting order for notes is not being saved properly when the user switches between "cards" and "notes" mode. The current implementation uses a single sorting configuration for both cards and notes.

### Step 4: Implement the Solution
To fix this, we need to store the sorting configuration separately for cards and notes. We can achieve this by modifying the `SortType` enum and the `CardBrowser` class.

### Step 5: Modify the `SortType` Enum
We need to modify the `save` method in `SortType` to store the sorting configuration in a way that distinguishes between cards and notes.

fun fromCol(config: Config, preferences: SharedPreferences, cardsOrNotes: CardsOrNotes): SortType {
    return when (cardsOrNotes) {
        CARDS -> fromColForCards(config, preferences)
        NOTES -> fromColForNotes(config, preferences)
    }
}

private fun fromColForCards(config: Config, preferences: SharedPreferences): SortType {
    // existing implementation for cards
}

private fun fromColForNotes(config: Config, preferences: SharedPreferences): SortType {
    val colOrder = config.get<String>("sortType")
    val type = entries.firstOrNull { it.ankiSortType == colOrder } ?: NO_SORTING
    val sortBackwards = config.get<Boolean>("browserNoteSortBackwards") ?: false
    if (type == SORT_FIELD && preferences.getBoolean("cardBrowserNoSortingNotes", false)) {
        return NO_SORTING
    }
    return type.also { it.mOrderAsc = sortBackwards }
}
