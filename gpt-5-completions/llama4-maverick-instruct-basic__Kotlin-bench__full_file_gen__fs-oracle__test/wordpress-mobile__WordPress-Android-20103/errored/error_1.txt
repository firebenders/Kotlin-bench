To address the issue of bringing back the Discover Onboarding for new users, we need to modify the `ReaderDiscoverViewModel` to show the onboarding screen when the user has no followed tags and is not a logged-in user (i.e., a new user). 

Here's how you can achieve this:

[start of WordPress/src/main/java/org/wordpress/android/ui/reader/discover/ReaderDiscoverViewModel.kt]
```kotlin
class ReaderDiscoverViewModel @Inject constructor(
    // ... existing code ...
    private val meStore: MeStore,
    // ... existing code ...
) : ScopedViewModel(mainDispatcher) {
    // ... existing code ...

    private fun observeDiscoverFeed() {
        // listen to changes to the discover feed
        _uiState.addSource(readerDiscoverDataProvider.discoverFeed) { posts ->
            launch {
                val userTags = getFollowedTagsUseCase.get()
                val isLoggedIn = meStore.hasAccessToken()
                if (userTags.isEmpty() && !isLoggedIn) {
                    _uiState.value = DiscoverUiState.EmptyUiState.ShowWelcomeOnboardingUiState {
                        parentViewModel.onShowReaderInterests()
                    }
                } else if (userTags.isEmpty()) {
                    _uiState.value = DiscoverUiState.EmptyUiState.ShowNoFollowedTagsUiState {
                        parentViewModel.onShowReaderInterests()
                    }
                } else {
                    // ... existing code ...
                }
            }
        }
    }

    // ... existing code ...

    sealed class DiscoverUiState(
        // ... existing code ...
    ) {
        // ... existing code ...

        sealed class EmptyUiState : DiscoverUiState(fullscreenEmptyVisibility = true) {
            // ... existing code ...

            data class ShowWelcomeOnboardingUiState(override val action: () -> Unit) : EmptyUiState() {
                override val titleResId = R.string.reader_discover_empty_title
                override val subTitleRes: Int = R.string.reader_discover_onboarding_subtitle
                override val buttonResId = R.string.reader_discover_onboarding_button_text
                override val illustrationResId = R.drawable.illustration_reader_onboarding
            }

            // ... existing code ...
        }
    }
}
```

In the above code, we've added a new `EmptyUiState` called `ShowWelcomeOnboardingUiState` to `ReaderDiscoverViewModel`. This state is shown when the user has no followed tags and is not logged in. We've also added the necessary string resources for the onboarding screen.

With these changes, the Discover Onboarding screen will be shown to new users who have no followed tags, providing a better user experience.
